# rm for Windows

Windows 向けに実装した `rm` です。  
POSIX.1-2017 と GNU `rm` の使い勝手をできるだけ保ちつつ、`cmd.exe` や PowerShell からでも Unix 系シェルに近い感覚で使えることを目指しています。

## 特徴

- `-f`, `-i`, `-r`, `-R` などの基本的な `rm` オプションをサポート
- `-I`, `-v`, `-d`, `--preserve-root`, `--no-preserve-root` などの GNU 拡張を一部サポート
- Windows で削除前に読み取り専用属性を外してから削除
- `.` と `..` をオペランドとして拒否
- ルート削除を既定で保護
- Windows 専用の内部 glob 展開

## Windows 向けの挙動

Windows の `cmd.exe` や PowerShell は、Unix 系シェルのように `*.txt` などの glob を native exe に渡す前に自動展開しません。  
このツールはその差を埋めるため、必要な場合だけ内部で glob 展開を行います。

挙動は次のとおりです。

- まず、各オペランドをリテラルなパスとして扱います
- そのパスが存在しない場合のみ、内部で glob 展開を試みます
- glob は Windows 向けに大文字小文字を区別しません
- glob にマッチしない場合は、元の文字列をそのまま削除対象として扱います

つまり、`*` `?` `[` を含む名前のファイルが実在する場合は、そのリテラル名を優先します。

## ビルド

```powershell
cargo build --release
```

生成物:

```text
target\release\rm.exe
```

## 使い方

```text
rm [オプション]... [ファイル]...
```

例:

```powershell
rm file.txt
rm -r dir
rm -rf dir
rm -i *.txt
rm -I file1 file2 file3
rm -v file1 file2
```

## 主なオプション

- `-f`, `--force`
  存在しないファイルや引数を無視し、確認なしで削除します。
- `-i`
  削除前に毎回確認します。
- `-I`
  3 つ以上のファイル、または再帰削除の前に 1 回だけ確認します。
- `-r`, `-R`, `--recursive`
  ディレクトリを再帰的に削除します。
- `-d`, `--dir`
  空のディレクトリを削除します。
- `-v`, `--verbose`
  削除した対象を表示します。
- `--preserve-root`
  ルート削除を禁止します。既定値です。
- `--no-preserve-root`
  ルート削除保護を無効にします。

## 終了ステータス

- `0`: 正常終了
- `1`: 削除エラー
- `2`: オプションエラー

## 注意

- `-r` を付けずにディレクトリを削除しようとするとエラーになります
- `rm -rf` は非常に強力です。対象をよく確認してから実行してください
- `--one-file-system` オプションは現在フラグとして受け付けますが、実動作は未実装です

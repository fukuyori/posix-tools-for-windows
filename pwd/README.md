# pwd

Windows 向けに実装した `pwd` コマンドです。  
POSIX `pwd` の `-L` / `-P` の考え方をできるだけ保ちつつ、Windows のファイルシステムに合わせて大文字小文字を区別せずに動作します。

## 目的

このツールは、Windows 上でも次のような挙動を目指しています。

- `-P` では実際の物理パスを表示する
- `-L` では `PWD` 環境変数を優先し、論理パスを維持する
- `PWD` が現在ディレクトリと同じ場所を指しているかを検証する
- Windows では大文字小文字を区別しない
- 必要に応じて内部で glob を使い、既存のパス要素の表記を実ファイルシステムに近づける

## ビルド

```powershell
cargo build --release
```

## 使い方

```powershell
pwd [オプション]
```

### オプション

- `-L`
  `PWD` 環境変数を優先して論理パスを表示します。
- `-P`
  シンボリックリンクなどを解決した物理パスを表示します。
- `--help`
  ヘルプを表示します。
- `--version`
  バージョン情報を表示します。

`-L` と `-P` を同時に指定した場合は、最後に書かれた方が有効です。  
オプションを指定しない場合は `-P` と同じ動作です。

## 動作の考え方

### `-P`

`std::fs::canonicalize` を使って物理パスを取得し、Windows の verbatim path prefix (`\\?\`) を取り除いて表示します。

### `-L`

`PWD` を使うのは、次の条件をすべて満たす場合だけです。

- 絶対パスである
- 実在するディレクトリである
- `.` や `..` を含まない
- 現在の作業ディレクトリと同じ場所を指している

条件を満たさない場合は `-P` 相当の結果にフォールバックします。

## Windows 固有の扱い

- パス比較は大文字小文字を無視します
- `PWD` が有効な場合、各パス要素を case-insensitive に探索して、既存エントリの表記へ寄せます
- この探索には `glob` クレートを利用しています

たとえば `PWD` が `C:\users\name\project` でも、実際のディレクトリエントリが `C:\Users\name\project` なら、その表記に近づけて返します。

## テスト

```powershell
cargo test
```

現在は主に次をテストしています。

- `PWD` に `.` や `..` が含まれる場合の拒否
- glob 用のメタ文字エスケープ

## 制限

- Windows 専用実装です
- POSIX 完全互換ではありません
- Windows のシンボリックリンクやジャンクションの扱いは、`std::fs::canonicalize` の結果に依存します
- `-L` の論理パス維持は `PWD` が適切に設定されていることを前提にしています

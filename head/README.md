# head - Rust実装のファイル先頭表示ツール

`head` は POSIX 標準の `head` コマンドの Rust 実装です。ファイルまたは標準入力の先頭部分を表示します。Windows での動作を想定し、内部 glob 展開で Linux 同等の挙動を実現しています。

## 機能

### POSIX 標準機能

- **`-n, --lines=[-]NUM`** - 先頭 NUM 行を表示（デフォルト: 10）
  - 負の値で末尾 NUM 行を除いて表示
  - 旧形式 `-NUM` にも対応
  
- **`-c, --bytes=[-]NUM`** - 先頭 NUM バイトを表示
  - 負の値で末尾 NUM バイトを除いて表示
  - サイズサフィックス対応: `b`, `KB`, `K`, `MB`, `M`, `GB`, `G`

- **複数ファイル処理** - ファイルごとにヘッダーを表示

### GNU 拡張機能

- **`-q, --quiet, --silent`** - ヘッダーを表示しない
- **`-v, --verbose`** - 単一ファイルでもヘッダーを表示
- **`-z, --zero-terminated`** - 行区切りを NULL 文字に変更

### 文字コード対応

自動判定により以下の文字コードに対応：
- **UTF-8** （BOM 検出対応）
- **Shift_JIS**
- **EUC-JP**

### Windows 向け glob 展開

Windows 環境では、シェルがワイルドカードを展開しないため、内部で自動的に glob パターンを展開します：

- **`*.txt`** - 全 .txt ファイルを展開
- **`file?.txt`** - 単一文字マッチにより file1.txt, file2.txt などを展開
- **大文字小文字を区別しない** - Windows ファイルシステムの仕様に準拠
- **パス区切り文字対応** - バックスラッシュ（`\`）を含むパターンも処理
- **相対パス対応** - `..\subdir\*.txt` などのパターンをサポート

マッチしたファイルは自動的に昇順にソートされます。パターンがマッチしない場合は元のパターンをそのまま渡し、適切なエラーを表示します。

## 使用例

```bash
# 先頭10行を表示（デフォルト）
head file.txt

# 先頭20行を表示
head -n 20 file.txt
head -20 file.txt          # 旧形式

# 末尾5行を除いて表示
head -n -5 file.txt

# 先頭100バイトを表示
head -c 100 file.txt

# 先頭1024バイトを表示
head -c 1K file.txt

# 複数ファイルを表示（ヘッダー付き）
head file1.txt file2.txt

# ヘッダーなしで表示
head -q file1.txt file2.txt

# 常にヘッダーを表示
head -v single_file.txt

# 標準入力から読み込み
echo "test" | head -n 1

# glob パターンで複数ファイルを処理 (Windows)
head -n 5 *.txt
head -n 5 src\*.rs
```

## ビルド

```bash
cargo build --release
```

実行ファイルは `target\release\head.exe` に生成されます。

## サイズサフィックス

`-c`/`--bytes` オプションで使用可能なサフィックス：

| サフィックス | 換算値 |
|------------|--------|
| `b` | 512 |
| `kB` / `KB` | 1,000 |
| `K` | 1,024 |
| `MB` | 1,000,000 |
| `M` | 1,048,576 |
| `GB` | 1,000,000,000 |
| `G` | 1,073,741,824 |

例：`head -c 10MB file.bin` は先頭 10,000,000 バイトを表示します。

## 実装詳細

### glob 展開の仕組み

入力クリーニング：
1. 標準入力セパレータ（`-`）はそのまま渡す
2. `*` または `?` を含むパターンは glob パターンとして扱う
3. ワイルドカード未使用のパターンはそのまま渡す

処理：
- glob ライブラリで `case_sensitive: false` を指定（Windows 仕様）
- マッチしたファイルパスを収集して返す
- ディレクトリはフィルタリングし、ファイルのみを対象

エラー時：
- glob パターン解析エラー時は元のパターンを返す
- マッチなしの場合も元のパターンを返す

### 文字コード判定アルゴリズム

1. BOM チェック（EF BB BF） → UTF-8
2. UTF-8 として有効か検証 → UTF-8
3. Shift_JIS スコアと EUC-JP スコアを計算
4. スコアが高い方を採用（デフォルト: Shift_JIS）

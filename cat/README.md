# cat

Rust で実装した `cat` です。Windows での実用性を重視しつつ、POSIX `cat` の基本動作にできるだけ寄せています。

## 方針

- 基本は入力バイト列をそのまま標準出力へ連結します
- Windows では `stdin` / `stdout` をバイナリモードに設定します
- `-u` は Windows で厳密な意味付けまでは追わず、動作すればよいものとして扱います
- GNU 互換の便利オプションも一部サポートしています

## 確認済みの互換範囲

現在、次の挙動はテストで確認しています。

- 不明オプションは診断を出して非 0 で終了する
- 引数なしなら標準入力を読む
- `-` オペランドはその位置で標準入力を読む
- 入力ファイルにエラーがあっても残りの入力を処理し、最終的に非 0 で終了する
- 通常のファイル入力はテキスト変換せず、バイト列をそのまま出力する
- `-u` 指定時は、少なくとも raw copy 経路で即時出力される

関連テストは [tests/posix.rs](/d:/home/source/rust/tools/cat/tests/posix.rs) にあります。

## 未保証の点

次の点は、まだ「完全な POSIX 準拠」としては保証していません。

- 診断メッセージの文言や形式の細部
- 特殊ファイルやシグナル周りの挙動
- POSIX テストスイート相当での網羅的検証
- `-u` の全環境での厳密な意味の一致

そのため、この実装は現時点では「POSIX の基本動作に近い Windows 向け `cat`」という位置づけです。

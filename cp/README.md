# cp

Rust で実装した `cp` コマンドです。  
Windows 上でも POSIX / GNU `cp` に近い操作感になるよう、`glob` 展開を内部で行います。

## 現在の状態

日常利用をかなり意識した水準まで実装済みです。  
特に Windows で使う際に困りやすい、ワイルドカード展開や再解析ポイントの扱いを重点的に整えています。

## 主な対応内容

- `*` `?` `[]` を使った内部 glob 展開
- `-R`, `-r`
- `-f`, `-i`, `-n`
- `-p`
- `-H`, `-L`, `-P`
- `-a`, `-u`, `-v`, `-b`
- `-t`, `-T`
- 日本語のヘルプメッセージとエラーメッセージ

## Windows での挙動

- PowerShell / `cmd.exe` が自動展開しないワイルドカードは、`cp` 側で展開します
- `-P` 指定時は、シンボリックリンクに加えてジャンクションなどのディレクトリ再解析ポイントもリンクとして複製します
- 同一ファイル判定は Windows のファイル ID ベースで行います
- `-p` は通常ファイルの時刻保持を行います
- ディレクトリの属性保持は、Windows の制約上、安全側で一部省略することがあります
- シンボリックリンク作成に失敗した場合は、管理者権限または開発者モード不足の可能性が分かる日本語エラーを返します

## まだ割り切っている点

- GNU `cp` と完全に同一の互換性を保証するものではありません
- Windows の再解析ポイントは種類が多いため、特殊なケースでは差異が出る可能性があります
- ACL や所有者など、Windows 固有の詳細属性までは保持していません

## テスト

以下で確認できます。

```powershell

```

- glob 展開

# `sed` for Windows

Windows 上で使うことを前提にした `sed` 実装です。  
POSIX `sed` にできるだけ近い挙動を目指しつつ、Windows で使いやすい差分は明示的に残しています。

通常の `sed` 利用:

- 単純な置換、抽出、削除
- `-n`, `p`, `d`
- 基本的なアドレス範囲
- `-E`
- `-i`
- `N`, `P`, `D`
- `b`, `t`, `T`
- `r`, `w`

このあたりは、日常用途でかなり実用的に使える状態を目標に整備しています。

## 方針

- デフォルトの正規表現は POSIX BRE 寄り
- `-E` / `-r` で ERE 寄り
- 複数入力ファイルはデフォルトで 1 本の連続ストリームとして処理
- Windows では入力ファイル引数に対して内部で glob 展開
- Windows では既存パス解決を大文字小文字非区別で実施

## 実装済みの主な互換ポイント

- BRE/ERE の切り替え、`\<` `\>`（単語境界）
- BRE の bracket expression を意識した変換
- パターン内の後方参照
- 置換文字列内の `&`, `\1`〜`\9`, `\&`, `\\`, `\U \L \u \l \E`（大小変換）
- `s///Ng`（N 番目以降をすべて置換）、`s///w FILE`
- 空正規表現 `//` による直前パターンの再利用
- アドレス: `$`, `/re/`, `/re/I`, `/re/M`, `first~step`, `0,/re/`, `addr,+N`, `addr,~N`
- `c` は範囲アドレスで最終行に一度だけテキストを出力（POSIX）
- `N`, `P`, `D` を使う複数行パターンスペース（`D` はサイクル全体を再開）
- ブロック内外への `b` / `t` / `T` 分岐（コマンド列をフラット化して実行）
- `q [code]` / `Q [code]`（`-i` でも書き戻してから終了コードを返す）
- `a` / `i` / `c` の行末 `\` による複数行テキスト
- `l [width]`（GNU 互換の折り返し、`-l N` / `--line-length=N`）
- `r` / `R` / `w` / `W` / `F` / `e` / `z` / `v` コマンド
  （`w` は起動後最初の書き込みで truncate、`/dev/stdout` `/dev/stderr` 対応）
- スクリプト先頭の `#n` による自動出力抑制（POSIX）
- `--expression=` / `--file=` / `--posix` / `--sandbox` / `--debug`
- 正規表現・分岐先ラベルの実行前検証（エラーを黙って握りつぶさない）
- POSIX 文字クラス (`[[:digit:]]` など)
- 未閉じ区切りや未知フラグなどの基本的な構文エラー検出

## Windows 固有の挙動

- `*`, `?`, `[]` を内部で glob 展開します
- glob は大文字小文字を区別しません
- 既存ファイルのリテラル指定も大文字小文字を区別せず解決します
- glob は POSIX シェル寄りに、`*` がディレクトリ境界をまたがず、先頭 `.` は明示時のみマッチします

## Git Bash / MSYS 利用時の注意

Git Bash / MSYS2 環境では、コマンドライン引数のうち `/` で始まるものを
シェルが Windows パスへ自動変換します（POSIX path conversion）。
このため `'/^#!/d'` のような sed スクリプトはバイナリに届く時点で
`C:\Program Files\Git\^#!\d` のような文字列に書き換えられ、
`不明なコマンド: 'C'` のようなエラーになります。

回避策は次のいずれかです:

- 環境変数で無効化: `MSYS_NO_PATHCONV=1 sed -n -e '/^#!/d' ...`
- 先頭スラッシュを 2 つにする: `sed -n -e '//^#!/d' ...`
- `cmd.exe` / PowerShell から直接呼ぶ（自動変換は発生しません）

本実装はエラー時に該当する `-e` 番号と位置、そして上記ヒントを出力します。

## GNU / 独自拡張

- `-E`, `-r`, `-s`, `-u`, `-z`, `-l N`, `--sandbox`, `--posix`
- `-i[SUFFIX]`（サフィックスは結合形式のみ、GNU と同じ）
- `Q`, `T`, `z`, `R`, `W`, `F`, `e`, `v`
- `first~step`, `0,/re/`, `addr,+N`, `addr,~N`, `/re/I`, `/re/M`
- 置換の `\U \L \u \l \E`

## GNU との既知の差分（v2.2.0 での変更点）

- 以前のバージョンにあった `-i backup`（次引数をサフィックスとして解釈する BSD 風の形式）は
  GNU 互換のため廃止しました。`sed -i 's/a/b/' file` が GNU と同じように動きます
- `y` の `-` はリテラルです（以前の tr 風範囲展開は廃止、GNU と同じ）

## 既知の制限

- 完全な POSIX 準拠を保証するものではありません
- 正規表現エンジンは Rust の crate ベースなので、細かな互換差分は残る可能性があります
- エラーメッセージ文言は GNU `sed` と完全一致ではありません
- GNU `sed` や各 Unix 実装の端ケースをすべて再現しているわけではありません

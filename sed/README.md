# `sed` for Windows

Windows 上で使うことを前提にした `sed` 実装です。  
POSIX `sed` にできるだけ近い挙動を目指しつつ、Windows で使いやすい差分は明示的に残しています。

通常の `sed` 利用:

- 単純な置換、抽出、削除
- `-n`, `p`, `d`
- 基本的なアドレス範囲
- `-E`
- `-i`
- `N`, `P`, `D`
- `b`, `t`, `T`
- `r`, `w`

このあたりは、日常用途でかなり実用的に使える状態を目標に整備しています。

## 方針

- デフォルトの正規表現は POSIX BRE 寄り
- `-E` / `-r` で ERE 寄り
- 複数入力ファイルはデフォルトで 1 本の連続ストリームとして処理
- Windows では入力ファイル引数に対して内部で glob 展開
- Windows では既存パス解決を大文字小文字非区別で実施

## 実装済みの主な互換ポイント

- BRE/ERE の切り替え
- BRE の bracket expression を意識した変換
- パターン内の後方参照
- 置換文字列内の `&`, `\1`, `\&`, `\\`
- 空正規表現 `//` による直前パターンの再利用
- `N`, `P`, `D` を使う複数行パターンスペース
- `q [code]` / `Q [code]`
- POSIX 文字クラス (`[[:digit:]]` など)
- `a`, `i`, `c`, `r`, `w`, `b`, `t`, `T`, `:` の基本文法
- 未閉じ区切りや未知フラグなどの基本的な構文エラー検出

## Windows 固有の挙動

- `*`, `?`, `[]` を内部で glob 展開します
- glob は大文字小文字を区別しません
- 既存ファイルのリテラル指定も大文字小文字を区別せず解決します
- glob は POSIX シェル寄りに、`*` がディレクトリ境界をまたがず、先頭 `.` は明示時のみマッチします

## GNU / 独自拡張

- `-E`, `-r`
- `-i[SUFFIX]`
- `-s`
- `-u`
- `-z`
- `Q`
- `T`
- `z`
- `first~step`

## 既知の制限

- 完全な POSIX 準拠を保証するものではありません
- 正規表現エンジンは Rust の crate ベースなので、細かな互換差分は残る可能性があります
- エラーメッセージ文言は GNU `sed` と完全一致ではありません
- GNU `sed` や各 Unix 実装の端ケースをすべて再現しているわけではありません
